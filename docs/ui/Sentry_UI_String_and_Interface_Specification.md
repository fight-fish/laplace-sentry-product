
# Sentry UI String and Interface Specification
### Sentry UI 字串與介面規範書

- **Document Status**: Official
- **Version**: v2.1
- **Language Policy**: UI 顯示以中文為主，必要時附英文輔助
- **Purpose**: 作為 Sentry UI 之唯一正式字串、介面語意、互動顯示與日誌翻譯規範，防止新舊功能混淆、字串漂移與互動語意失真。

---

# 0. Scope 文件適用範圍

本文件適用於：

- `Frontend/src/tray/tray_app.py`
- 與 UI 顯示直接相關之 Adapter 回傳轉譯層
- Dashboard / Eye / Intent Menu / Toast / Dialog / Logs 等所有使用者可見文字
- 後端原始日誌之 UI 白話轉譯顯示

本文件不處理：

- 後端 CLI 原始訊息設計
- 資料契約 Schema
- 錯誤碼制度
- 系統治理層規範

上述內容分別以 Volume C / D / E 等文件為準。

# 1. Core UI Principles 核心介面原則

## 1.1 Single Wording Authority
所有使用者可見字串，均應以本文件為唯一正式依據。  
程式碼不得任意發明近義字、縮寫字或替代說法。

## 1.2 Chinese-First Display
介面顯示以中文為主。  
英文僅允許出現在下列情境：

- 括號輔助理解
- 專有名詞
- 技術除錯資訊
- 規格文件內對照用途

## 1.3 Intent Clarity
所有按鈕、通知、對話框、日誌翻譯，必須優先表達「使用者現在能理解的意圖」，而非內部技術狀態。

## 1.4 No Legacy Drift
舊版可沿用之文字必須明確標示為 Legacy 保留資產；  
新 UI 不得因重構而造成同一功能在不同畫面出現不同說法。

# 2. UI View System 視圖系統定義

本系統 UI 分為兩個主要視圖：

- **View A**：The Eye（哨兵之眼）
- **View B**：Dashboard（控制台）

---

# 3. View A — The Eye
### 哨兵之眼（預設畫面）

程式啟動後的預設畫面，採極簡風格，負責：

- 顯示主圖示
- 接收拖曳輸入
- 進入意圖判斷
- 快速切往控制台

---

## 3.1 View A 顯示文字規範

| 元件位置 | 顯示文字 | 備註 |
|---|---|---|
| 主圖示 Tooltip | `Sentry：拖曳檔案至此以新增或更新` | 滑鼠移到眼睛上時顯示 |
| 右下角按鈕 | `哨兵管理` | 切換至控制台 |
| 按鈕 Tooltip | `開啟詳細控制台` | 滑鼠移到按鈕上時顯示 |

---

## 3.2 View A 行為語意規則

1. 預設進入本畫面。
2. 本畫面不得顯示過多目錄樹工作區管理欄位。
3. 本畫面是「入口」與「意圖判斷層」，不是完整管理頁。
4. 所有拖曳操作若需後續選擇，必須進入 Intent Menu 或 Interactive Dialog。

---

# 4. Intent Menu
### 意圖選單

拖曳放開後彈出的智慧選單，用於判斷使用者當前意圖。

---

## 4.1 Intent Menu 項目規範

| 項目 | 選單文字 | 觸發條件 |
|---|---|---|
| 選項 1 | `🛡️ 加入新哨兵 (New Project)` | 拖入的是資料夾 |
| 選項 2 | `⚡ 單次更新 (One-time Update)` | 拖入的是檔案 |
| 取消 | `(點擊外部自動取消)` | 無需顯示按鈕 |

---

## 4.2 Intent Menu 行為規則

1. 選單屬於暫時性互動層。
2. 點擊外部即取消，不需額外「取消」按鈕。
3. 同一拖曳事件不得同時觸發多個意圖。
4. 文案必須強調「行為意圖」，而非技術實作。

---

# 5. Global Toast
### 全域狀態通知

懸浮氣泡通知，全面取代舊有底部狀態列。

---

## 5.1 Toast 類型規範

| 類型 | 範本格式 | 範例 |
|---|---|---|
| 處理中 | `⏳ 正在處理：{task_name}...` | `⏳ 正在掃描專案...` |
| 成功（綠） | `✔ {action} 成功` | `✔ 專案新增成功` |
| 失敗（紅） | `✖ {action} 失敗：{reason}` | `✖ 新增失敗：路徑不存在` |
| 資訊（灰） | `ℹ️ {message}` | `ℹ️ 已取消操作` |

---

## 5.2 Toast 原則

1. Toast 應短、清楚、可立即理解。
2. Toast 不承載長篇教學內容。
3. 失敗類 Toast 必須指出具體原因。
4. 同一動作不可同時出現相互矛盾的 Toast。

---

# 6. View B — Dashboard
### 控制台（Legacy 保留區）

本畫面基於 v1.8 改造，保留列表管理功能。  
屬於較完整的管理界面，而非極簡入口。

---

## 6.1 Dashboard 顯示文字規範

| 元件位置 | 顯示文字 | 備註 |
|---|---|---|
| 左上角導航 | `↩ 返回` | 切換回 View A |
| 視窗標題 | `Sentry 控制台` | 固定標題 |
| 專案列表表頭 | `UUID, 專案名稱, 監控狀態, 模式` | 與 v1.8 一致 |
| 狀態欄位 | `監控中, 已停止` | 固定用詞 |
| 模式欄位 | `靜默, 互動` | 固定用詞 |

---

## 6.2 Dashboard 原則

1. Dashboard 為管理中心。
2. Legacy 保留項目不得任意改名。
3. 新功能若進入 Dashboard，必須符合既有命名風格。
4. 控制台不應回退成技術測試頁外觀。

---

# 7. Context Menu
### 右鍵選單

保留 v1.8 的管理功能，作為專案列操作主入口。

---

## 7.1 右鍵動作規範

| 動作 | 選單文字 |
|---|---|
| 手動更新 | `🔄 立即手動更新 (Manual Update)` |
| 修改 | `📝 修改專案...` |
| 刪除 | `🗑️ 刪除此專案...` |

---

## 7.2 Context Menu 規則

1. 動作用詞應以動作導向為主。
2. 保留符號圖示以提高辨識性。
3. 刪除動作必須保留省略號或後續確認流程。
4. 不得在同一層選單混入與專案無關的全域設定。

---

# 8. Detail & Bottom
### 詳情與底部區塊

用於資訊顯示、忽略規則編輯與日誌瀏覽。

---

## 8.1 區塊規範

| 區塊 | 內容 | 變動說明 |
|---|---|---|
| 詳情面板 | `名稱 / 類型 / 路徑鍵 / 目前複製來源 / 複製作用範圍 / 註解` | 保留 v1.8 詳情資訊方向，補入目錄樹複製語意 |
| 忽略說明 | `忽略設定區...` | 完全保留 |
| 功能按鈕 | `編輯目錄樹忽略規則…`、`複製目錄樹` | 補入目錄樹複製操作 |
| 日誌瀏覽器 | `哨兵日誌 (Live Logs)` | 新增，取代舊輸入區 |

補充規則：

1. 當使用者選取目錄樹節點時，詳情面板除名稱、類型、路徑鍵、註解外，應額外顯示：
   - 目前複製來源
   - 按下「複製目錄樹」後的實際作用範圍

2. 當選取為專案根節點時，詳情面板應明確顯示：
   - `目前複製來源：整個專案根目錄`

3. 當選取為資料夾或檔案節點時，詳情面板應明確顯示：
   - `目前複製來源：<path_key>`

4. 「複製目錄樹」採單鍵智慧判斷模式：
   - 選取專案根節點時，複製整個專案樹
   - 選取資料夾節點時，複製該資料夾子樹
   - 選取檔案節點時，複製該檔案節點的最小結構

5. 詳情面板與成功提示不得只顯示模糊語句（例如：`目前目錄樹`），必須可直接判讀複製來源。

---

## 8.2 日誌瀏覽器規格

1. 顯示風格：黑底白字
2. 角色定位：唯讀日誌視窗
3. 不得混入手動輸入功能
4. 必須支援時間軸分組邏輯

---

# 9. Interactive Dialogs
### 互動對話

View A 在特殊情境下觸發的引導對話。

---

## 9.1 對話規範

| 情境 | 標題 | 內容摘要 |
|---|---|---|
| 飢渴模式（Hunting） | `收到資料夾` | `已暫存專案路徑：{path}... 找不到預設的 readme.md。👉 請現在「直接拖曳」您想寫入的 Markdown 檔案進來！` |
| 引導失敗 | `新增失敗 - 需要手動修正` | `快速新增失敗... 已為您切換至【控制台】，請在下方手動輸入路徑。` |

---

## 9.2 對話框原則

1. 對話框必須表達下一步行動。
2. 不得只顯示錯誤而不給方向。
3. 引導文案需偏「陪同操作」，而非技術拒絕。

---

# 10. Log Translator
### 日誌翻譯字典

將後端原始日誌轉換為白話文顯示。

---

## 10.1 翻譯表

| 原始關鍵字 | 顯示圖示 | 顯示文字（HTML） | 顏色代碼 |
|---|---|---|---|
| 哨兵啟動 | 👁️ | `哨兵已就位，開始監控` | `#00FFFF` |
| Stopping | 💤 | `哨兵已暫停值勤` | `#888888` |
| created | ✨ | `發現新檔案` | `#00FF00` |
| modified | 📝 | `偵測到變更` | `#FFFFFF` |
| deleted | 🗑️ | `檔案已移除` | `#FF5555` |
| 智能靜默 | 🛡️ | `觸發過熱保護（進入靜默模式）` | `#FFFF00` |
| 成功觸發更新 | ✅ | `正在執行目錄樹更新...` | `#44AAFF` |
| BLACKLIST | 🔒 | `安全機制：已自動排除輸出檔監控` | `#555555` |

---

## 10.2 翻譯規則

1. 翻譯後文字應以使用者理解為優先。
2. 不得逐字照抄後端技術訊息。
3. 同一原始關鍵字必須映射到固定顯示文字。
4. 顏色語意必須保持一致。

---

# 11. Timeline Specification
### 時間軸規範

定義日誌在 UI 中的時間分組顯示方式。

---

## 11.1 格式規範

| 元件 | 格式 | 範例 |
|---|---|---|
| 日期標題 | `📅 YYYY-MM-DD` | `📅 2025-12-04` |
| 單行時間 | `HH:MM:SS` | `14:30:05` |

---

## 11.2 顯示規則

1. 跨日期日誌必須分組。
2. 同一日期下顯示單行時間。
3. 時間軸排序採自然時間順序。
4. 日期標題屬視覺分段，不視為日誌事件本體。
